Stage 1: HTML and Web Basics

Programming is very literal. You have to tell the computer exactly what to do so that you get the results you want.

HTML is a type of document for web content. It contains a Title, Header and Body structure.

The body of an HTML document is made up of text that has markup language. This text can be inline or block. Block places a box around the text
much like a text box drawn in Word or Power point.

Void tags,such as break, do not require closing tags whereas other tags such as the one used for italics requires a closing tag with an / at the end of the text. If a closing tag is not used where appropriate,
then everything after the opening tag will take on the same style or attribute and there will be no terminating point.

"Boxes Boxes Boxes!!!" Breaking down pages into boxes to be identified, sized, and characterized is an important step to web development.

Think of HTML as the house or structure, CSS as the styling or decor or architecture of the house, and Java Script as the Interactive components that can change the attributes of the structure.

DOM is a reference to the Tree Structure of an HTML document. It is hierarchical in nature and HTML creates the elements of the DOM.

CSS is the styling-- Cascading Stying Sheets-- used to apply to HTML documentation. It uses the concept of inheritance where descendant elements inherit text related properties of ascendant elements. This, however, does not apply with box related properties.

CSS allows you to assign attributes to elements of your document in a number of ways. For example by setting font, font size, color, background color, etc. It also allows you to control the size of the boxes in your document.

You can assign those attributes in groups using class selectors such as "<div class=h1>". This allows the user to ascribe styling to everying that is in the header elements

There are two recommended ways to include CSS in your HTML document.

  • Write CSS in the "<head>" of your HTML
  • Link your HTML to a separate CSS file

Setting box sizes can be tricky. There are two ways to do this.

  • Set sizes in terms of percents rather than pixels.
  • Set the box-sizing attribute to border-box for every element.
Divs by default will take up the entire width of the page. Using display: flex; will allow the divs to appear side by side

Images should include an alt to describe what should appear if the browser prohibits the image from appearing .

Code, Test, Refine are important steps to creating your web page. Looking for natural boxes, repeated styles and semantic elements, write your HTML and apply styles from biggest to smallest.

It is also important to avoid repetition by using CSS. This is achieved by using CSS styles to apply to sections throughout the document such as title, header, and text. This ensures document consistency and allows you to easily add to or update HTML without repetitively ascribing specific attributes throughout the document

Developer Tools are a nifty way to view you site, code and changes at the same time. Also use these links to verify your page complies with HTML and CSS standards.Click here: To verify HTML and here: To verify CSS

Stage 4: Let Anyone Modify Your Webpage

ANetwork is a group of entities that can communicate even though they are not directly connected. Things that make up a network are message and bits, routers, and rules for deciding who gets to use the resources (QoS)

Latency measures the time (milliseconds) it takes for a message to move from the source to destination in a network. Bandwidth is the amount of information that can be transmitted per unit of time (bps). cnet internet speed test measures the bandwidth you're getting over your internet.

Adding more nodes and adding more bits are ways to reduce latency and increase number of things you can distinguish respectively.

Protocol is set of rules that tell you how 2 entities can talk to each other, like a client and server. Client uses a protocol -- HTTP -- to send a message. The server gets the message, runs code, finds file, runs code and sends back a response.

HTML documents - contain a DOM structure: doctype; opening and closing HTML tags; the head which contains title, meta- data, java script, css cscript, etc.; and the body tag which contains the content of what appears on the webpage.

URL or Uniform Resource Locator is made up of protocol (http://), host (www.udacity.com), and path.

Query Parameter is in the URL such as the Get parameter. Example would be like "/foo?p=1&q=neat". "?" is used for the first parameter, "&" is used to separate for every parameter after. Name = value.

"#fragment" is next in the URL path. It exists in the browser only and not sent to server. when there are query parameters, fragment follows.

Port - to make an internet connection in a URL you need the address of a machine and a port. Between "localhost" and "path" is the port. Port default =80. "http://localhost:8000/" is typical.

HTTP which is the main protocol of the web. Request from your browser begins with a request line. Request line is made up of Method - Path - Version such as "GET" (used to get a document) and "POST", the path which includes up through parameters but does not include the fragment, and version which is usually HTTP/1. 1 or 1.0. Note that the path is a relative path. You could also specify the full path which would start with the http link through to the path and parameters.So the host is used for making the connection and the path is used for making the request.

Request link also includes headers (name: value). "Host" as in "Host: www.example.com" which is required in 1.1. Webservers may have multiple names so this is necessary. Next header is "User- Agent:" as in "User-Agent= chrome" which is who is making the request (i.e., what type of machine). User agent important because it helps you control who has access and how they can use your site. Always use User-Agents appropriately when using your web software.

HTTP Responses look similar to request line. Status line (HTTP/1.1) - status code (200 which is ok, or 302 found or 404 not found or 500 server error) - Reason phrase is the Status code verbiage such as ok, found, not found, server error. Status line is followed by a number of headers. Some are required, or assumed. "Date:", "Server:" which should be hidden, "Content- Type:" like text/html, "Content-Length".

Servers - purpose is to respond to Client request. 2 types which are Static or pre-written file like an image; and Dynamic where response built on the fly by program that is running. Most content today is dynamic and built by programs called web applications which is what we'll be building.

HTML Forms - allow you to create an entry space using the tag form. Input is the tag that can take a couple of attributes. Like input name. When press enter or click submit, you are submitting a form. URLs can't have spaces in theme so your browser will enter spacers such as better+flight+search. User "password" input type to collect passwords but will not be secure. Use "checkbox" with name to display checkbox. Use "radio" and give same name so can only select one option. "label" attribute lets you label each box.

Validation allows you to make your website safe from hackers using input validation. Validation means verifying on the server side so it knows when it receives junk or bad data. It also ensures it is clear to the user when incorrect info is entered.

Limiting values that the user can select doesn't protect against someone using a form with bad data to post that bad data to your server. You have to have a way to verify the validity of the input and send error with error message if invalid.

String substitution - Use to avoid repetition such as if we want to return a lot of bold content without continually using bold tags. So instead use "%s" % . Escaping will prevent data entered from messing up our html . Such as <>. You can also import cgi in python which will apply the escape command where needed. Redirection: Used to allow you to share success link and reload page. Directs user to a success page. Also reloading the success page doesnt resubmit the form. and to have distinct pages for forms and successes

HTML Templates - Separate different type of code, make more readable code, make a more secure website, and make html easier to modify. Use Template libraries like jinja2 to build complicated strings.

Variable Substitution - in templates allows you to use variables to populate values in an HTML document using tags. Statement Syntax and Loop Syntax are methods for achieving this kind of substitution.

Template Inheritance - use one template for setting up the page, and subtemplates that redefine the content. Again demonstrates the concept of abstraction in that it minimizes code to avoid repetition by projecting a template format for the subtemplates containing content.
